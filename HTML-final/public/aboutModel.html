About the Model

Learn about how we constructed this 3D structural visualization of an E. coli cell and how we set it on display

Abstract

No complete, three-dimensional reconstruction of a prokaryote-scale cell exists to represent proteomic, metabolomic, and genomic data in an accessible manner. With the availability of experimental and computational information on the structures, stoichiometry, and localization of cellular components of Escherichia coli, along with the development of visualization tools such as cellPACK and chromosome simulation models, we have the “ingredients” we need to put together a model for the well-studied organism. This project integrates this data into an interactive, three-dimensional structural biology model of an E. coli cell for modern web browsers. The cell model contains still structural representations of the proteins, protein complexes, cell membrane, and chromosome, and is built on a pipeline that can produce a similar model from different data. It can be used as a stepping stone for organizing the information available on E. coli and as a stepping stone to develop models for other organisms at the prokaryotic and eukaryotic scales.

Constructing the Model

+ Expand all

  • Structures

    Genome-scale models (GEMs) are used for computational studies of metabolism, protein synthesis, and transcriptional regulation based on a specific organism. Recently, structural information has been mapped to those models, which are now called genome-scale models with protein structures (GEM-PROs) and provide a new view linking systems and structural biology. This pipeline has expanded knowledge of biochemical network properties and can help make predictions on whole-cell phenotypes. This comprehensive information can only be visualized in two dimensions, so it is necessary to create a pipeline that generates three-dimensional models using GEM-PRO output data. One way to generate these models is by gathering the data for the system and putting it into a single “recipe” file (will be discussed further in the model assembly) and using this recipe to generate a virtual model through a computational engine with built-in assembly algorithms.

  • Orientations and Localizations

    For membrane proteins, orientation data was provided by the Orientations of Proteins in Membranes (OPM) database and the tool TMHMM, which uses hidden Markov model (a machine learning algorithm) to predict transmembrane helices of membrane proteins. For cytoplasmic proteins and membrane proteins without detailed data, the SWISS-MODEL homology-modeling tool was found to be extraordinarily useful to predict morphologies. cellPACK includes these orientation constraints when packing species onto points on the generated membrane, described below.

  • Stoichiometries

    Stoichiometries of proteomics were primarily based off experimental data from Schmidt et al., 2016.

    Complex concentrations or copy numbers of complexes per cell were estimated using mass fractions from the ME model. The copy numbers were calculated with the assumption that there is a "limiting" complex and that the number of complexes were to be maximized. In other words, the mass fraction gave us the ratio between the complexes and we wanted to figure out how to best maximize the number of complexes while maintaining the ratio. However, because some of the complexes were severely limiting such that less than 1% of proteins would be in complexes, another complex would be used as the "limit" instead. The code for this calculation can be found on Github.

Cell Membranes were made using Blender software. E. coli tomograms (add reference and image here) were first used to create a representation of the membranes. The tomograms were processed using IMOD to create a 3D representation of the membrane. However, due to the lack of full cell tomograms for E. coli, experimental dimensions (listed below) were used to reconstruct an estimated membrane representation. The E. coli cell was assumed to be a volume composed of a cylinder with two hemishperes attached at the ends.The model from the tomograms was then shrink wrapped onto the cylindrical model to give the overall model a more realistic shape. Additionally, the estimated model was able to fill in a lot of the incomplete regions of the tomogram derived membranes.

The model is scaled such that 1 Angstrom = 1 cm. The E. coli cell model here is approximately 2.5 micrometers long by 1.2 micrometers wide. The general width of the periplasm around the middle of the cell (DPB) is about 30-25 nm and the widths at the ends, PPC1 and PPC2, are 260nm and 80nm, respectively. The experimental dimensions used were referenced from (heinemanna and schmidt paper put here) for E. coli MG1655 in LB media.

This model uses a 500 bpb (beads per base pair) representation of the E. coli chromosome. Dimensions are about 1.78 micrometers by 0.78 micrometers with the origin of representation at midcell. The midcell origin was chosen arbitrarily. The DNA chromosome model was generously shared with us by William C. Hacker and Professor Adrian H. Elcock.
